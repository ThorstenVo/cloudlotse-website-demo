# Sticky Navigation and Process Line Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep the eazy.cloud navigation reachable during scrolling and turn the case-study step boxes into a clearly non-interactive process line.

**Architecture:** Extend the existing React components with semantic state and markup hooks, while keeping all responsive presentation in the existing `mobile.css` layer. The header owns only whether the page has scrolled; CSS owns the compact visual state. The case study owns an ordered list whose labels and supporting copy come from the existing locale catalog.

**Tech Stack:** React 18 JSX compiled by esbuild, plain CSS, Node.js built-in test runner, jsdom prerendering.

## Global Constraints

- Preserve the established eazy.cloud colors, typography, spacing, locale switching, and mobile hamburger.
- Add no dependencies and refactor no unrelated sections.
- The process component is informational and uses semantic ordered-list markup.
- Respect `prefers-reduced-motion`.
- Do not deploy as part of implementation; deployment requires a separate confirmed step.

## File Map

- `ui_kits/website/Chrome.jsx`: owns scroll state and exposes `.is-scrolled` on the existing top bar.
- `ui_kits/website/Chapters.jsx`: renders the semantic three-station process line.
- `ui_kits/website/translations.mjs`: stores localized titles and supporting copy for all stations.
- `mobile.css`: owns sticky-header presentation, compact dimensions, reduced motion, anchor offsets, and horizontal/vertical process-line layouts.
- `tests/site-structure.test.mjs`: regression coverage for source structure, behavior hooks, semantic markup, and responsive CSS.
- `ui_kits/website/dist/*.js`, `en/index.html`, `de/index.html`, `index.html`: generated by `npm run build`; never edit manually.

- [ ] **Step 3: Add minimal scroll state to `Chrome.jsx`**

Inside `TopBar`, add:

```jsx
const [scrolled, setScrolled] = React.useState(false);

React.useEffect(() => {
  const onScroll = () => setScrolled(window.scrollY > 16);
  onScroll();
  window.addEventListener("scroll", onScroll, { passive: true });
  return () => window.removeEventListener("scroll", onScroll);
}, []);
```

Change the header opening tag to:

```jsx
<header
  className={"cl-topbar" + (scrolled ? " is-scrolled" : "")}
  style={{ zIndex: 30, inset: "0 0 auto", height: "var(--topbar-height)", color: "#fff", borderBottom: "1px solid rgb(255 255 255 / 24%)" }}
>
```

Change the mobile menu inline inset from `76px 0 0 0` behavior to the CSS-owned top value by retaining only its existing `display: "none"` inline style.
