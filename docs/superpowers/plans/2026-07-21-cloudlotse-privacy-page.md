# CloudLotse Privacy Page and Consent Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish an English Privacy page and a self-hosted consent interface that prevents every Google request until a visitor explicitly accepts analytics.

**Architecture:** Keep the current static GitHub Pages layout and the existing React prototype components. Add a small esbuild pipeline that produces local browser scripts, a framework-independent consent module shared by the homepage and legal pages, and a static Privacy page. Analytics configuration is fail-closed: the repository ships with no container ID and therefore cannot load Google; once the real GTM ID is supplied, the same consent gate loads it only after acceptance.

**Tech Stack:** Static HTML/CSS, React 18.3.1, esbuild, browser JavaScript, Node.js built-in test runner, jsdom.

## Global Constraints

- First-release copy is English only.
- The Privacy page covers public website visitors and email enquiries only.
- Production hosting is standard ALL-INKL webhosting in Germany.
- Website contact uses email links only; there is no contact form or newsletter.
- Fonts, React, scripts, and ordinary website assets must be served locally.
- No external embeds, external CDNs, chat widgets, advertising tags, or additional marketing trackers.
- Google Analytics 4 is delivered through Google Tag Manager only after affirmative analytics consent.
- Use basic consent mode: before consent or after rejection, no request is sent to Google, including cookieless pings.
- Necessary consent storage is always available; analytics is optional.
- Accept and reject actions must be equally accessible and visually comparable.
- Rejecting analytics must not reduce website functionality.
- A permanent `Privacy settings` link must allow consent withdrawal or revision.
- GA4 user-level event retention must be configured to two months in the Google property.
- Ordinary non-contractual email enquiries are deleted after 12 months, subject to statutory retention duties.
- Treat the legal copy as a carefully sourced draft requiring qualified legal review before production launch.

```js
import { build, transform } from "esbuild";
import { mkdir, readFile, writeFile } from "node:fs/promises";
import path from "node:path";

const root = process.cwd();
const sourceDir = path.join(root, "ui_kits", "website");
const outDir = path.join(sourceDir, "dist");
const jsxSources = ["Chrome.jsx", "Hero.jsx", "Chapters.jsx", "Closing.jsx", "tweaks-panel.jsx", "app.jsx"];

await mkdir(outDir, { recursive: true });

await build({
  entryPoints: [path.join(sourceDir, "vendor-entry.js")],
  outfile: path.join(outDir, "vendor.js"),
  bundle: true,
  minify: true,
  platform: "browser",
  format: "iife",
  target: ["es2020"],
});

for (const filename of jsxSources) {
  const source = await readFile(path.join(sourceDir, filename), "utf8");
  const compiled = await transform(source, {
    loader: "jsx",
    jsx: "transform",
    format: "iife",
    minify: true,
    target: "es2020",
  });
  await writeFile(path.join(outDir, filename.replace(/\.jsx$/, ".js")), compiled.code);
}
```

- [ ] **Step 4: Implement consent persistence and the one-way GTM gate**

Create `privacy/consent.js` as a browser IIFE. Use JSON validation that accepts only version `1`, a boolean `analytics`, and a parseable ISO timestamp. Implement `setAnalyticsConsent` by storing the decision first and calling `loadGtm()` only for `true`. Implement `loadGtm()` with all of these guards:

```js
const STORAGE_KEY = "cloudlotse_privacy_consent_v1";
const CONSENT_VERSION = 1;

function loadGtm() {
  const id = window.CloudLotseAnalyticsConfig?.gtmContainerId;
  if (!/^GTM-[A-Z0-9]+$/.test(id || "")) return;
  if (document.querySelector(`script[data-cloudlotse-gtm="${id}"]`)) return;
  window.dataLayer = window.dataLayer || [];
  window.dataLayer.push({ "gtm.start": Date.now(), event: "gtm.js" });
  const script = document.createElement("script");
  script.async = true;
  script.dataset.cloudlotseGtm = id;
  script.src = `https://www.googletagmanager.com/gtm.js?id=${encodeURIComponent(id)}`;
  document.head.append(script);
}
```

On startup, load GTM only if a validated stored decision has `analytics: true`. Never load GTM for undecided or rejected states. `withdrawConsent()` removes the storage record and calls `window.location.reload()` so an already loaded third-party runtime is cleared.
